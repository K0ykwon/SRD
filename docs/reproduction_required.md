# Required Reproduction Bundle

This repository now exposes two explicit reproduction paths for the required synthetic benchmark items:

- Delayed KV
- Needle
- Delayed Copy
- sufficiency ablation with a lambda sweep
- score and aggregate implementation audit

The two paths serve different purposes:

- `run_reproduction_required.sh`: small, fast reproducibility bundle for logic and artifact validation
- `run_reproduction_required_longctx.sh`: paper-facing long-context bundle at `~15M` and `~50M` scale
- `run_reproduction_required_small_8k.sh`: `~50M` only, `8k`-context follow-up bundle

## Commands

Run the required synthetic benchmark bundle:

```bash
bash scripts/run_reproduction_required.sh
```

This runs:

- the three required synthetic tasks
- the main comparison variants
- a `sufficiency_loss_weight` sweep over `0.0, 0.05, 0.1, 0.25, 0.5`

Outputs are written under:

- `outputs/reproduction/required_synthetic/`

Run the larger long-context reproduction bundle:

```bash
bash scripts/run_reproduction_required_longctx.sh
```

This runs:

- the same three required synthetic tasks
- `transformer_full`, `srd_refresh`, `srd_refresh_sufficiency`, and `srd_refresh_sufficiency_detail`
- `compact` and `small` backbones
- contexts `1024`, `2048`, and `4096`
- a `sufficiency_weight` ablation over `0.0, 0.05, 0.1, 0.25, 0.5`

Outputs are written under:

- `outputs/reproduction/required_longctx/`

Run the `small`-only `8k` follow-up bundle:

```bash
bash scripts/run_reproduction_required_small_8k.sh
```

This runs:

- the same three required synthetic tasks
- `transformer_full`, `srd_refresh`, `srd_refresh_sufficiency`, and `srd_refresh_sufficiency_detail`
- `small` backbone only
- context `8192`
- a `sufficiency_weight` ablation over `0.0, 0.05, 0.1, 0.25, 0.5`

Outputs are written under:

- `outputs/reproduction/required_small_8k/`

Run the score and aggregate audit:

```bash
bash scripts/run_reproduction_audit.sh
```

Outputs are written under:

- `outputs/reproduction/audit/`

## Long-Context Bundle Size

The long-context bundle is intentionally much larger than the small smoke path.

- main runs: `216`
- sufficiency ablation runs: `180`
- total runs: `396`

Its model and context axes are:

- `compact`: shared backbone target `~15M`
- `small`: shared backbone target `~50M`
- contexts: `1024`, `2048`, `4096`

Use `MAX_RUNS=<n>` with `scripts/run_reproduction_required_longctx.sh` for debug truncation.

The default long-context shell wrapper now targets the current workstation's `16GB` class GPU budget:

- `micro_batch_size = 1`
- `bf16`
- reduced checkpoint cadence relative to the original Set A main preset

Required long-context suites now start from `srd_refresh_sufficiency_detail` first, followed by the simpler refresh variants and then the dense baseline. This makes the early finished cells more relevant to the active SRD question.

For long detached runs, prefer the detached launch wrappers under `scripts/` together with suite resume via existing per-run JSON files. The suite runner now supports `--skip-existing` so interrupted launches can resume from the current output directory instead of restarting the whole matrix.

The `8k` follow-up bundle size is:

- main runs: `36`
- sufficiency ablation runs: `45`
- total runs: `81`

## What The Audit Covers

The audit checks two things.

### 1. Score implementation

It runs deterministic perfect-prediction score checks for:

- `delayed_kv`
- `needle`
- `delayed_copy`

and records:

- which primary metric name each task uses
- whether perfect predictions score `1.0`
- which task metrics are emitted

### 2. Aggregate implementation

It writes deterministic mock results through the existing artifact writers and checks that:

- `aggregate_results.csv` is the flat per-run export
- `aggregate_grouped.csv` computes grouped mean/std over numeric fields
- the grouped metric mean/std matches a known expected value
